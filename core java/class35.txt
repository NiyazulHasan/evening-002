Q) Write a java program to find out length of a string without using predefined method i.e length()?

{

Q) Write a java program to display reverse of a string?

input:
	hello

output:
	olleh 

ex:
---
class Test  
{
	public static void main(String[] args) 
	{
		String str="hello";
		
		String rev="";
		for(int i=str.length()-1;i>=0;i--)
		{
			rev+=str.charAt(i);
		}
		System.out.println(rev);
	}
}

{

	{
		String str="racar";

Q) Write a java program to encode the string?

input:
	1106
output:
	AAJF 

class Test  
{
	public static void main(String[] args) 
	{
		String str="1106";
		
		for(int i=0;i<str.length();i++)
		{
			int n = Character.getNumericValue(str.charAt(i));
			if(n>0)
			{
				System.out.print((char)('A'+n-1));
			}
			else
			{
				int k = Integer.parseInt(str.substring(i-1,i+1));
				System.out.print((char)('A'+k-1));
			}
		}
	}
}

Q) Write a java program to display the string in a given format?

input:
	A1B2C3D4
output:
	ABBCCCDDDD 

class Test  
{
	public static void main(String[] args) 
	{
		String str="A1B2C3D4";
		
		for(int i=0;i<str.length();i++)
		{
			if(Character.isAlphabetic(str.charAt(i)))
			{
				System.out.print(str.charAt(i));
			}
			else
			{
				int k = Character.getNumericValue(str.charAt(i));
				for(int j=1;j<k;j++)
				{
					System.out.print(str.charAt(i-1));
				}
			}
		}
	}
}

StringBuffer
============
If our content change frequently then it is never recommanded to use String object because for every change a new object will be created.

To overcome this limitation Sun Micro System introduced StringBuffer.

In StringBuffer all the required changes will be done in a same object only.

constructor
----------

StringBuffer sb = new StringBuffer()
-------------------------------------
It creates empty StringBuffer object with default initial capacity of 16.

If capacity reaches to maximum capacity then new capacity will be created with below formulea.

ex:
	new capacity = current_capacity + 1 * 2;


class Test  
{
	public static void main(String[] args) 
	{
		StringBuffer sb = new StringBuffer();
		System.out.println(sb.capacity()); // 16
		
		sb.append("abcdefghijklmnop");
		System.out.println(sb.capacity()); // 16
		
		sb.append("qr");
		System.out.println(sb.capacity()); // 16+1*2 = 34
	}
}

Q) Write a java program to multiply two arrays?

input:
	2 6 1 
	8 3 
output:
	21663 (261*83)


class Test  
{
	public static void main(String[] args) 
	{
		int[] arr1={2,6,1}; 
		int[] arr2={8,3}; 
		int a = Integer.parseInt(arrayToString(arr1));
		int b = Integer.parseInt(arrayToString(arr2));
		System.out.println(a*b);
	}
	public static String arrayToString(int[] arr)
	{
		StringBuffer sb = new StringBuffer();
		for(int i : arr)
		{
			sb.append(i); //261   //83
		}
		return sb.toString();
	}
}

Q) Write a java program to count number of 2's?

input:
	22

output:
	6 (2,12,20,21,22)


class Test  
{
	public static void main(String[] args) 
	{
		int n=22;
		StringBuffer sb = new StringBuffer();
		for(int i=1;i<=n;i++)
		{
			sb.append(i);
		}
		
		int cnt=0;
		for(int i=0;i<sb.length();i++)
		{
			char ch = sb.charAt(i);
			if(ch=='2')
			{
				cnt++;
			}
		}
		System.out.println(cnt);
	}
}

StringBuilder 
=============
StringBuilder is exactly same as StringBuffer with following differences.

StringBuffer				StringBuilder
------------				--------------
All methods are synchronized.		No method is synchronized.

At a time only one thread is allowed	Multiple threads are allowed to operate 
to operate on StringBuffer. Hence 	on StringBuilder . Hence StringBuilder	
StringBuffer is thread safe.		is not thread safe.

Waiting time of a thread will increase	There is no waiting threads effectively 
effectively performance is low.		performance is high.

It is introduced in 1.0 version.	It is introduced in 1.5 version.


Q) Write a java program to display reverse of a given number?

input:
	123
output:
	321


class Test  
{
	public static void main(String[] args) 
	{
		int n = 123;
		
		String str = Integer.toString(n);	
		
		StringBuilder sb = new StringBuilder(str);
		
		System.out.println(sb.reverse().toString());
	}
}

Note:
-----
If our content not change frequently then we need to use String.

If our content change frequently where thread safety is required then we need to use StringBuffer.

If our content change frequently where thread safety is not required then we need to use StringBuilder.
